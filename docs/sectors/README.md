## Regra de uso

- Documento mestre: visão executiva, posicionamento do produto, arquitetura e prioridades.
- Documentos por setor: fluxos, responsabilidades, dados, APIs e riscos específicos da área.
- Documentos especializados antigos: manter apenas quando ainda forem úteis tecnicamente; marcar como suporte, legado ou arquivo quando deixarem de refletir a prioridade atual.
